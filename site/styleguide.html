Design System

The living reference for this site. Every swatch and component below is rendered straight from assets/style.css, and token values are read from the live stylesheet at load, so this page cannot drift from the real design system. If a swatch looks wrong here, the token is wrong everywhere.

Color tokens

Defined in :root. Backgrounds below are set with var(--token); the hex shown is the computed value read from the page, so it always matches what ships.

Vivid vs. ink. The saturated tokens (--pos, --neg, --amber) are for fills and marks — bars, chart dots, swatches. As small text they fail WCAG AA, so text and pills use the darker --pos-ink / --neg-ink / --amber-ink variants (all ≥ 4.5:1 on white and on their soft backgrounds).

Still intentionally local: the validated categorical chart palettes in map.html / trajectory.html (self-contained, CVD-checked sets) and bare #fff fills in SVG chart chrome.

Scale & foundations

Non-color primitives, all applied app-wide. Every font-size and every margin/padding/gap now resolves to a --text-* or --space-* token — no ad-hoc rem values remain, and analysis/audit_scale.py (wired into the test suite) fails the build if one creeps back. Ramp steps were tuned to the app's actual usage clusters, so snapping every value onto them moved nothing more than ~2px. SVG chart text (px) and a few relative em heading margins stay literal on purpose.
