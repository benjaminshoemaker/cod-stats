Community Consensus Methodology

This page explains how fan/community rankings are collected and converted into title rankings and career rollups. It is not a site-authored ranking and does not include official awards, objective stats, or resume metrics.

Scope Sources Per-title scoring Career rollup Era selector Caveats

Scope

Community Consensus summarizes how available fan/community sources ranked players during or after each title. It is a compilation layer. It should never be presented as "our top players" without the source context.

Source Tiers

Aggregate survey

Highest-value evidence. Examples: annual r/CoDCompetitive Top 30 surveys or large community poll projects with visible sample or methodology notes.

Thread cluster

Multiple ranked community ballots inside one discussion. Ballots are averaged before thread-level weighting so one thread does not behave like many independent sources.

Single ballot

One clean ranked community list. Useful for sparse older titles, but low weight and clearly labeled.

Per-Title Scoring

Each source rank becomes normalized points, then source weight adjusts for source kind, sample size, timing, and season coverage.

source_contribution = rank_points * source_weight * timing * season_coverage

Career Rollup

After each title has a consensus rank, the career score converts each included title rank to top-heavy points.

title_points = max(0, (31 - consensus_rank) / 30) ^ 2.5

Every selected title has the same maximum value. Better source coverage in modern titles affects confidence and within-title ranks, not the cross-title value of the season.

Era Selector

The Community Consensus page uses the same Eras control as the main leaderboard. Presets and custom title selections recompute the rollup from the selected titles, including totals, normalized views, event wins, source traces, and top-1/top-3/top-5/top-10 counts.

Caveats

Missing evidence is not zero evidence. Older titles have weaker source inventories, so rows must keep confidence, source counts, and source links visible. Blanks should be labeled as unknown or not found in the current inventory.
